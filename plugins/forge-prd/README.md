# forge-prd 插件

从任何原料通过强势拷问锻造出一份清晰 PRD，输出 MD 文件。

## 核心理念

**锻造，不是打磨**。输入可以是粗略想法、截图、PDF、HTML 或 MD，输出是一份结构化的 PRD 文件。需求本身可能就不对——插件会像红队一样强势质疑，帮你想清楚再做。

## 适用场景

- 只有一条粗略想法，还没形成完整需求
- 有截图、原型、PDF 等多模态素材，需要整合成结构化 PRD
- 需求方向本身不确定，需要通过强势质疑来验证合理性
- 希望最终输出一份可直接用于研发的 MD 格式 PRD 文件

## 4 阶段流程

1. **吸收**：读取所有素材（MD/PDF/HTML/图片），图片主动解读后请用户确认
2. **理解确认**：向用户复述"我理解你想要的是 X"，确认大方向正确
3. **三层拷问**：
   - 模糊层 → 边界层 → 假设层（强势质疑）
   - 每层结束后用户确认才进入下一层
4. **输出**：整合为 MD 文件，预览后写入 `<原名>.forged.md`

## 输出格式

文件命名：输入是单个文件时写到同目录，文件名为 `<原文件名>.forged.md`；输入是目录时写到该目录，文件名为 `PRD.forged.md`；输入来自对话时询问用户目标路径。

文档结构（灵活 + 最小骨架）：

**最小骨架（缺一不可）：**

- **问题描述** — 这个需求要解决什么问题？背景是什么？
- **目标** — 做完之后要达到什么效果？可衡量的成功标准是什么？
- **需求条目** — 具体的功能点 / 行为要求，逐条列出
- **约束与边界** — 技术约束、业务规则、边界条件、异常处理规则

**按需扩展（复杂需求自动加入，简单需求不加）：**

- 用户故事、非功能需求、验收标准、依赖、待确认项

## 使用方式

```
/forge-prd <需求素材文件/目录路径>
```

路径支持：单个文件、目录、或省略（从当前目录自动扫描）。
