# forge-prd 插件

从任何原料通过强势拷问锻造出一份清晰 PRD，输出 MD 文件。

## 核心理念

**锻造，不是打磨**。输入可以是粗略想法、截图、PDF、HTML 或 MD，输出是一份结构化的 PRD 文件。需求本身可能就不对——插件会像红队一样强势质疑，帮你想清楚再做。

## 与 dev-flow-lite:grill 的区别

| | forge-prd（本插件） | dev-flow-lite:grill |
|---|---|---|
| 使用时机 | 需求原料不完整/方向不明 | 需求已清晰，准备编码前 |
| 目标 | 从原料锻造出 PRD 文件 | 确认设计决策，不产出文件 |
| 拷问强度 | 强势，红队风格 | 决策树遍历，推荐优先 |
| 最终产出 | `xxx.forged.md` 文件 | 无 |

## 4 阶段流程

1. **吸收**：读取所有素材（MD/PDF/HTML/图片），图片主动解读后请用户确认
2. **理解确认**：向用户复述"我理解你想要的是 X"，确认大方向正确
3. **三层拷问**：
   - 模糊层 → 边界层 → 假设层（强势质疑）
   - 每层结束后用户确认才进入下一层
4. **输出**：整合为 MD 文件，预览后写入 `<原名>.forged.md`

## 使用方式

```
/forge-prd <需求素材文件/目录路径>
```

路径支持：单个文件、目录、或省略（从当前目录自动扫描）。
